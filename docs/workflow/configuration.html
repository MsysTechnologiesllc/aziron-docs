Configuration Guide

Comprehensive guide to configuring Aziron's workflow engine, including global settings, workflow-specific configurations, and environment management.

Overview‚Äã

Aziron's configuration system provides:

  • Hierarchical Configuration: Global, workflow, and step-level settings
  • Environment Management: Development, staging, and production configurations
  • Dynamic Updates: Runtime configuration changes without restarts
  • Validation: Comprehensive configuration validation and error reporting
  • Security: Encrypted configuration for sensitive values

Configuration Hierarchy‚Äã

Configuration Precedence‚Äã

Configuration values are resolved in the following order (highest to lowest priority):

  1. Runtime Parameters - Values passed during execution
  2. Environment Variables - OS environment variables
  3. Workflow Configuration - Workflow-specific settings
  4. Global Configuration - System-wide defaults
  5. Built-in Defaults - Hardcoded fallback values

Configuration Management‚Äã

Loading Configuration‚Äã

// Load configuration from multiple sources
config, err := LoadConfig(ConfigOptions{
    GlobalConfigPath:   "/config/global.json",
    EnvironmentPath:    "/config/environments/",
    WorkflowConfigPath: "/config/workflows/",
    Environment:        "production",
    SecretsProvider:    vault.NewProvider(),
})
if err != nil {
    log.Fatalf("Failed to load configuration: %v", err)
}

Environment Variables‚Äã

Use environment variables to override configuration values:

# System configuration
export AZIRON_LOG_LEVEL=debug
export AZIRON_DEBUG=true
export AZIRON_PORT=8080

# Database configuration
export AZIRON_DB_HOST=localhost
export AZIRON_DB_PASSWORD=secret123
export AZIRON_DB_MAX_CONNECTIONS=50

# Security configuration
export AZIRON_JWT_SECRET=your-jwt-secret
export AZIRON_AUTH_ENABLED=true

# Workflow engine configuration
export AZIRON_MAX_CONCURRENT_EXECUTIONS=100
export AZIRON_DEFAULT_TIMEOUT=300000

# Agent configuration
export AZIRON_AGENT_TIMEOUT=30000
export AZIRON_AI_MODEL=gpt-4
export AZIRON_AI_TEMPERATURE=0.7

Dynamic Configuration Updates‚Äã

# Update global configuration
curl -X PUT http://localhost:8080/api/v1/config/global \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "workflow_engine": {
      "max_concurrent_executions": 200
    }
  }'

# Update workflow configuration
curl -X PUT http://localhost:8080/api/v1/config/workflows/email_processor \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "config": {
      "timeout": 240000,
      "variables": {
        "ai_model": "gpt-4-turbo"
      }
    }
  }'

# Reload configuration
curl -X POST http://localhost:8080/api/v1/config/reload \
  -H "Authorization: Bearer YOUR_TOKEN"

Best Practices‚Äã

Configuration Design‚Äã

  1. Environment Separation: Use separate configurations for each environment
  2. Secrets Management: Never store secrets in plain text
  3. Validation: Always validate configuration before applying
  4. Documentation: Document all configuration options thoroughly

Performance Optimization‚Äã

  1. Caching: Cache frequently accessed configuration values
  2. Lazy Loading: Load configuration only when needed
  3. Minimal Updates: Only update changed configuration values
  4. Monitoring: Monitor configuration performance impact

Security Best Practices‚Äã

  1. Encryption: Encrypt sensitive configuration data
  2. Access Control: Implement role-based access to configuration
  3. Audit Logging: Log all configuration changes
  4. Regular Rotation: Rotate secrets and certificates regularly

Deployment Strategies‚Äã

  1. Rolling Updates: Apply configuration changes gradually
  2. Canary Deployments: Test configuration changes on subset of instances
  3. Rollback Plans: Have rollback procedures for configuration changes
  4. Health Checks: Monitor system health after configuration changes
