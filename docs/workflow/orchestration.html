Orchestration Modes

Aziron's workflow engine supports multiple orchestration strategies to coordinate agent responses and manage complex multi-agent interactions. Each mode offers different trade-offs between performance, intelligence, and complexity.

Overview​

Orchestration modes determine how the workflow engine handles and coordinates responses from multiple agents:

ModeDescriptionPerformanceIntelligenceUse Cases
noneNo orchestrationHighestNoneSimple workflows, direct responses
bufferResponse bufferingHighLowStreaming aggregation, simple collection
agentAI-powered orchestrationMediumHighestComplex reasoning, synthesis
hybridCombined buffering + AIMediumHighBalanced performance and intelligence
graphGraph-based optimizationVariableMediumComplex dependencies, optimization

None Mode (none)​

Overview​

No response orchestration is performed. Agent responses are returned directly as they arrive.

Configuration​

{
  "orchestration": {
    "mode": "none"
  }
}

Characteristics​

  • Performance: Highest - minimal overhead
  • Latency: Lowest - direct response forwarding
  • Intelligence: None - no processing or analysis
  • Memory Usage: Minimal - no response buffering

Use Cases​

  • Simple request-response workflows
  • Performance-critical applications
  • Real-time streaming requirements
  • Direct agent interactions

Example Workflow​

{
  "id": "simple_query",
  "orchestration": {
    "mode": "none"
  },
  "steps": [
    {
      "id": "ask_agent",
      "type": "message",
      "config": {
        "to_agent_id": "assistant",
        "content": "{{user_question}}"
      }
    }
  ]
}

Buffer Mode (buffer)​

Overview​

Collects and buffers streaming responses, returning complete responses when finished.

Configuration​

{
  "orchestration": {
    "mode": "buffer",
    "buffer_config": {
      "timeout": 30000,
      "max_size": 1048576,
      "completion_check_interval": 500,
      "response_timeout": 5000
    }
  }
}

Configuration Options​

  • timeout: Maximum time to wait for complete response (ms)
  • max_size: Maximum buffer size in bytes
  • completion_check_interval: How often to check for completion (ms)
  • response_timeout: Time to wait after last chunk received (ms)

Response Completion Detection​

The buffer mode uses several heuristics to detect complete responses:

  1. Natural Endings: Sentences ending with ., !, ?
  2. Mathematical Results: Expressions with = followed by numbers
  3. Inactivity Timeout: No new chunks for specified duration
  4. Explicit Markers: Special completion tokens

Use Cases​

  • Streaming response aggregation
  • Collecting multi-part responses
  • Simple response combination
  • WebSocket response handling

Example Workflow​

{
  "id": "streaming_analysis",
  "orchestration": {
    "mode": "buffer",
    "buffer_config": {
      "timeout": 60000,
      "response_timeout": 3000
    }
  },
  "steps": [
    {
      "id": "parallel_analysis",
      "type": "parallel",
      "config": {
        "parallel_steps": ["sentiment_analysis", "entity_extraction"]
      }
    }
  ]
}

Agent Mode (agent)​

Overview​

Uses a dedicated AI agent to intelligently orchestrate and synthesize responses from multiple agents.

Configuration​

{
  "orchestration": {
    "mode": "agent",
    "agent_config": {
      "orchestrator_agent_id": "orchestrator",
      "max_iterations": 5,
      "synthesis_model": "gpt-4",
      "enable_reasoning": true,
      "context_window": 8192
    }
  }
}

Configuration Options​

  • orchestrator_agent_id: ID of the orchestrating agent
  • max_iterations: Maximum orchestration rounds
  • synthesis_model: AI model for response synthesis
  • enable_reasoning: Enable step-by-step reasoning
  • context_window: Maximum context size for orchestrator

Orchestration Agent Capabilities​

The orchestration agent can:

  1. Analyze Responses: Understand context and intent
  2. Synthesize Information: Combine multiple agent outputs
  3. Reason About Next Steps: Plan additional actions
  4. Validate Results: Check for completeness and accuracy
  5. Generate Summaries: Create coherent final responses

Use Cases​

  • Complex multi-agent coordination
  • Intelligent response synthesis
  • Dynamic workflow adaptation
  • Advanced reasoning tasks

Example Workflow​

{
  "id": "research_synthesis",
  "orchestration": {
    "mode": "agent",
    "agent_config": {
      "orchestrator_agent_id": "research_coordinator",
      "max_iterations": 3,
      "enable_reasoning": true
    }
  },
  "steps": [
    {
      "id": "gather_sources",
      "type": "parallel",
      "config": {
        "parallel_steps": ["web_search", "database_query", "document_analysis"]
      }
    }
  ]
}

Hybrid Mode (hybrid)​

Overview​

Combines response buffering with AI agent validation and enhancement.

Configuration​

{
  "orchestration": {
    "mode": "hybrid",
    "hybrid_config": {
      "buffer_timeout": 30000,
      "agent_validation": true,
      "fallback_to_buffer": true,
      "enhancement_threshold": 0.7,
      "orchestrator_agent_id": "validator"
    }
  }
}

Configuration Options​

  • buffer_timeout: Buffering phase timeout
  • agent_validation: Enable AI validation of buffered responses
  • fallback_to_buffer: Fallback to buffer mode if agent fails
  • enhancement_threshold: Quality threshold for agent enhancement
  • orchestrator_agent_id: Agent for validation/enhancement

Processing Flow​

  1. Buffer Phase: Collect responses using buffer mode
  2. Validation Phase: AI agent validates response quality
  3. Enhancement Phase: Improve response if below threshold
  4. Fallback: Return buffered response if agent fails

Use Cases​

  • Balanced performance and quality
  • Quality assurance workflows
  • Robust production systems
  • Performance-sensitive intelligent applications

Example Workflow​

{
  "id": "customer_support",
  "orchestration": {
    "mode": "hybrid",
    "hybrid_config": {
      "buffer_timeout": 15000,
      "agent_validation": true,
      "enhancement_threshold": 0.8
    }
  },
  "steps": [
    {
      "id": "support_response",
      "type": "message",
      "config": {
        "to_agent_id": "support_agent",
        "content": "{{customer_inquiry}}"
      }
    }
  ]
}

Graph Mode (graph)​

Overview​

Advanced graph-based orchestration with topology analysis and execution optimization.

Configuration​

{
  "orchestration": {
    "mode": "graph",
    "graph_config": {
      "topology_analysis": true,
      "parallel_optimization": true,
      "dependency_validation": true,
      "convergence_tracking": true,
      "branch_coordination": true,
      "critical_path_analysis": true,
      "load_balancing": true,
      "adaptive_timeout": true
    }
  }
}

Configuration Options​

  • topology_analysis: Analyze workflow graph structure
  • parallel_optimization: Optimize parallel execution
  • dependency_validation: Validate dependency resolution
  • convergence_tracking: Track branch convergence points
  • branch_coordination: Coordinate parallel branches
  • critical_path_analysis: Identify critical execution paths
  • load_balancing: Balance load across parallel branches
  • adaptive_timeout: Adjust timeouts based on complexity

Graph Analysis Features​

Topology Analysis​

  • Execution Levels: Group steps by dependency depth
  • Parallel Branches: Identify independent execution paths
  • Convergence Points: Find where branches merge
  • Critical Path: Determine longest execution sequence

Optimization Features​

  • Parallel Scheduling: Optimize concurrent execution
  • Resource Allocation: Balance agent workload
  • Dependency Resolution: Ensure correct execution order
  • Deadlock Detection: Prevent circular dependencies

Graph Metrics​

The graph orchestrator collects detailed metrics:

{
  "graph_metrics": {
    "total_levels": 4,
    "max_parallelism": 6,
    "critical_path_length": 8,
    "graph_complexity": 0.75,
    "execution_efficiency": 0.92,
    "parallel_utilization": 0.88
  }
}

Use Cases​

  • Complex workflow optimization
  • High-concurrency applications
  • Performance-critical systems
  • Large-scale orchestration

Example Workflow​

{
  "id": "data_processing_pipeline",
  "orchestration": {
    "mode": "graph",
    "graph_config": {
      "topology_analysis": true,
      "parallel_optimization": true,
      "critical_path_analysis": true
    }
  },
  "steps": [
    {
      "id": "fetch_data",
      "type": "parallel",
      "config": {
        "parallel_steps": ["fetch_api_data", "fetch_db_data", "fetch_files"]
      }
    },
    {
      "id": "process_data",
      "type": "parallel", 
      "dependencies": ["fetch_data"],
      "config": {
        "parallel_steps": ["validate_data", "transform_data", "enrich_data"]
      }
    },
    {
      "id": "generate_reports",
      "type": "parallel",
      "dependencies": ["process_data"],
      "config": {
        "parallel_steps": ["summary_report", "detailed_report", "metrics_report"]
      }
    }
  ]
}

Orchestration Context​

Shared Context​

All orchestration modes have access to shared context:

{
  "shared_context": {
    "goal": "Process customer inquiry and generate response",
    "user_instruction": "Help the customer with their billing question",
    "task_decomposition": [...],
    "intermediate_results": {...},
    "agent_invocations": [...],
    "tool_invocations": [...],
    "facts_learned": {...},
    "execution_log": [...]
  }
}

Context Management​

  • Goal Tracking: Maintain primary workflow objective
  • Result Aggregation: Collect intermediate results
  • Execution History: Record all agent/tool invocations
  • Fact Learning: Accumulate discovered information
  • Session Management: Maintain workflow session state

Choosing the Right Mode​

Decision Matrix​

Use none when:

  • Simple request-response patterns
  • Performance is critical
  • No response coordination needed
  • Real-time streaming required

Use buffer when:

  • Streaming response aggregation needed
  • Simple response collection required
  • Moderate performance requirements
  • Basic completion detection sufficient

Use agent when:

  • Complex response synthesis needed
  • Intelligent coordination required
  • Quality over performance
  • Advanced reasoning necessary

Use hybrid when:

  • Balanced performance and intelligence needed
  • Quality assurance required
  • Fallback mechanisms desired
  • Production robustness important

Use graph when:

  • Complex workflow optimization needed
  • High concurrency requirements
  • Performance optimization critical
  • Advanced dependency management required

Best Practices​

Configuration Guidelines​

  1. Start Simple: Begin with none or buffer mode
  2. Profile Performance: Measure latency and resource usage
  3. Scale Gradually: Move to more complex modes as needed
  4. Monitor Metrics: Track orchestration effectiveness

Error Handling​

{
  "orchestration": {
    "mode": "hybrid",
    "error_handling": {
      "retry_count": 3,
      "fallback_mode": "buffer",
      "timeout_escalation": true
    }
  }
}

Resource Management​

  • Set appropriate timeouts for each mode
  • Configure memory limits for buffering
  • Monitor agent resource usage
  • Implement circuit breakers for reliability
