Built-in Workflows

Aziron includes several pre-configured workflows designed for common enterprise scenarios. These workflows demonstrate best practices and can be used as-is or customized for your specific needs.

Google Workspace Orchestrator‚Äã

Overview‚Äã

The Google Workspace Orchestrator is a comprehensive workflow that coordinates various Google Workspace operations including Gmail, Calendar, Drive, and Chat integrations.

Configuration‚Äã

{
  "id": "google_workspace_orchestrator",
  "name": "Google Workspace Agentic Orchestrator", 
  "description": "Main workflow that receives user prompts and orchestrates appropriate responses using Google Workspace tools",
  "version": "1.0.0",
  "orchestration": {
    "mode": "agent",
    "agent_config": {
      "orchestrator_agent_id": "orchestrator_agent",
      "max_iterations": 3
    }
  }
}

Workflow Steps‚Äã

1. Analyze Prompt‚Äã

{
  "id": "analyze_prompt",
  "name": "Analyze User Prompt",
  "type": "message",
  "config": {
    "from_agent_id": "system",
    "to_agent_id": "orchestrator_agent",
    "content": "{{user_prompt}}"
  },
  "on_success": ["determine_actions"],
  "timeout": 300000
}

2. Determine Actions‚Äã

{
  "id": "determine_actions", 
  "name": "Determine Required Actions",
  "type": "message",
  "config": {
    "from_agent_id": "orchestrator_agent",
    "to_agent_id": "action_planner_agent",
    "content": "Based on the analysis, determine the specific actions needed: {{step_analyze_prompt_response}}"
  },
  "dependencies": ["analyze_prompt"],
  "on_success": ["execute_workspace_actions"]
}

3. Execute Workspace Actions‚Äã

{
  "id": "execute_workspace_actions",
  "name": "Execute Google Workspace Actions", 
  "type": "parallel",
  "config": {
    "parallel_steps": [
      "workspace_chat_action",
      "workspace_email_action", 
      "workspace_calendar_action",
      "workspace_drive_action"
    ]
  },
  "dependencies": ["determine_actions"],
  "on_success": ["synthesize_response"]
}

4. Synthesize Response‚Äã

{
  "id": "synthesize_response",
  "name": "Synthesize Final Response",
  "type": "message",
  "config": {
    "from_agent_id": "action_planner_agent",
    "to_agent_id": "response_synthesizer_agent",
    "content": "Combine all workspace action results into a coherent response"
  },
  "dependencies": ["execute_workspace_actions"]
}

Usage Example‚Äã

curl -X POST http://localhost:8080/api/v1/workflows/google_workspace_orchestrator/execute \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "context": {
      "user_prompt": "Schedule a meeting with the marketing team next week and send them the project proposal",
      "user_id": "user123",
      "workspace_credentials": "..."
    },
    "started_by": "user123"
  }'

Email Processor‚Äã

Overview‚Äã

Processes incoming emails, analyzes content, and generates appropriate responses using AI agents.

Key Features‚Äã

  • Content Analysis: Extract key information from emails
  • Intent Detection: Understand the purpose of the email
  • Response Generation: Create contextually appropriate replies
  • Priority Classification: Categorize emails by urgency
  • Attachment Processing: Handle email attachments

Workflow Steps‚Äã

1. Parse Email‚Äã

{
  "id": "parse_email",
  "name": "Parse Email Content",
  "type": "function",
  "config": {
    "function_name": "parse_email_content",
    "parameters": {
      "email_data": "{{email_content}}"
    }
  },
  "on_success": ["analyze_content"]
}

2. Analyze Content‚Äã

{
  "id": "analyze_content",
  "name": "Analyze Email Content",
  "type": "parallel",
  "config": {
    "parallel_steps": [
      "extract_entities",
      "detect_intent", 
      "classify_priority",
      "sentiment_analysis"
    ]
  },
  "dependencies": ["parse_email"],
  "on_success": ["generate_response"]
}

3. Generate Response‚Äã

{
  "id": "generate_response",
  "name": "Generate Email Response",
  "type": "message",
  "config": {
    "from_agent_id": "email_analyzer",
    "to_agent_id": "response_generator",
    "content": "Generate appropriate response based on analysis: {{analysis_results}}"
  },
  "dependencies": ["analyze_content"]
}

Usage Example‚Äã

curl -X POST http://localhost:8080/api/v1/workflows/email_processor/execute \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "context": {
      "email_content": {
        "from": "customer@company.com",
        "subject": "Billing Question",
        "body": "I have a question about my recent invoice...",
        "attachments": []
      }
    },
    "started_by": "email_system"
  }'

Chat Responder‚Äã

Overview‚Äã

Handles real-time chat conversations with context awareness and intelligent response generation.

Key Features‚Äã

  • Context Maintenance: Track conversation history
  • Multi-turn Conversations: Handle complex dialogues
  • Intent Recognition: Understand user goals
  • Response Optimization: Generate engaging responses
  • Escalation Handling: Route complex queries to humans

Workflow Steps‚Äã

1. Process Message‚Äã

{
  "id": "process_message",
  "name": "Process Chat Message",
  "type": "message", 
  "config": {
    "from_agent_id": "user",
    "to_agent_id": "chat_processor",
    "content": "{{user_message}}",
    "metadata": {
      "conversation_id": "{{conversation_id}}",
      "message_history": "{{message_history}}"
    }
  },
  "on_success": ["analyze_intent"]
}

2. Analyze Intent‚Äã

{
  "id": "analyze_intent",
  "name": "Analyze User Intent",
  "type": "message",
  "config": {
    "from_agent_id": "chat_processor", 
    "to_agent_id": "intent_analyzer",
    "content": "Analyze intent for: {{step_process_message_response}}"
  },
  "dependencies": ["process_message"],
  "on_success": ["generate_chat_response"]
}

3. Generate Response‚Äã

{
  "id": "generate_chat_response",
  "name": "Generate Chat Response",
  "type": "conditional",
  "config": {
    "condition": "{{intent.complexity}} < 0.8",
    "true_steps": ["simple_response"],
    "false_steps": ["escalate_to_human"]
  },
  "dependencies": ["analyze_intent"]
}

Usage Example‚Äã

curl -X POST http://localhost:8080/api/v1/workflows/chat_responder/execute \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "context": {
      "user_message": "How do I reset my password?",
      "conversation_id": "conv_123",
      "user_id": "user456"
    },
    "started_by": "chat_system"
  }'

File Analyzer‚Äã

Overview‚Äã

Comprehensive file analysis workflow that processes uploaded files and extracts insights, metadata, and content.

Key Features‚Äã

  • Format Detection: Identify file types and formats
  • Content Extraction: Extract text and data from files
  • Metadata Analysis: Extract file properties and structure
  • Security Scanning: Check for security issues
  • Content Classification: Categorize file contents

Supported File Types‚Äã

  • Documents: PDF, DOCX, TXT, RTF
  • Spreadsheets: XLSX, CSV, TSV
  • Presentations: PPTX, PDF slides
  • Images: JPG, PNG, GIF, SVG
  • Data: JSON, XML, YAML
  • Code: Multiple programming languages

Workflow Steps‚Äã

1. File Validation‚Äã

{
  "id": "validate_file",
  "name": "Validate Uploaded File",
  "type": "function",
  "config": {
    "function_name": "validate_file",
    "parameters": {
      "file_path": "{{uploaded_file.path}}",
      "max_size": "{{max_file_size}}",
      "allowed_types": "{{allowed_file_types}}"
    }
  },
  "on_success": ["detect_format"],
  "on_failure": ["file_error"]
}

2. Format Detection‚Äã

{
  "id": "detect_format",
  "name": "Detect File Format",
  "type": "function",
  "config": {
    "function_name": "detect_file_format",
    "parameters": {
      "file_path": "{{uploaded_file.path}}"
    }
  },
  "dependencies": ["validate_file"],
  "on_success": ["parallel_analysis"]
}

3. Parallel Analysis‚Äã

{
  "id": "parallel_analysis",
  "name": "Parallel File Analysis",
  "type": "fork",
  "config": {
    "branches": [
      {
        "name": "content_extraction",
        "steps": ["extract_content", "analyze_content"]
      },
      {
        "name": "metadata_extraction", 
        "steps": ["extract_metadata", "classify_content"]
      },
      {
        "name": "security_analysis",
        "steps": ["virus_scan", "content_filter"]
      }
    ]
  },
  "dependencies": ["detect_format"],
  "on_success": ["merge_analysis_results"]
}

4. Merge Results‚Äã

{
  "id": "merge_analysis_results",
  "name": "Merge Analysis Results",
  "type": "join",
  "config": {
    "wait_for_branches": ["content_extraction", "metadata_extraction", "security_analysis"],
    "merge_strategy": "combine"
  },
  "dependencies": ["parallel_analysis"]
}

Usage Example‚Äã

curl -X POST http://localhost:8080/api/v1/workflows/file_analyzer/execute \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "context": {
      "uploaded_file": {
        "path": "/tmp/uploaded/document.pdf",
        "name": "financial_report.pdf",
        "size": 2048576
      },
      "analysis_options": {
        "extract_text": true,
        "generate_summary": true,
        "detect_entities": true
      }
    },
    "started_by": "user789"
  }'

Calendar Manager‚Äã

Overview‚Äã

Manages calendar operations including event creation, scheduling, conflict detection, and meeting coordination.

Key Features‚Äã

  • Event Creation: Create calendar events with participants
  • Conflict Detection: Identify scheduling conflicts
  • Availability Checking: Check participant availability
  • Meeting Coordination: Coordinate multi-participant meetings
  • Reminder Management: Set up event reminders

Workflow Steps‚Äã

1. Parse Request‚Äã

{
  "id": "parse_schedule_request",
  "name": "Parse Scheduling Request",
  "type": "message",
  "config": {
    "from_agent_id": "user",
    "to_agent_id": "calendar_parser",
    "content": "{{schedule_request}}"
  },
  "on_success": ["check_availability"]
}

2. Check Availability‚Äã

{
  "id": "check_availability",
  "name": "Check Participant Availability",
  "type": "parallel",
  "config": {
    "parallel_steps": ["check_organizer_availability", "check_participants_availability"]
  },
  "dependencies": ["parse_schedule_request"],
  "on_success": ["find_optimal_time"]
}

3. Find Optimal Time‚Äã

{
  "id": "find_optimal_time",
  "name": "Find Optimal Meeting Time",
  "type": "function",
  "config": {
    "function_name": "optimize_meeting_time",
    "parameters": {
      "availability_data": "{{availability_results}}",
      "preferences": "{{scheduling_preferences}}"
    }
  },
  "dependencies": ["check_availability"],
  "on_success": ["create_calendar_event"]
}

4. Create Event‚Äã

{
  "id": "create_calendar_event",
  "name": "Create Calendar Event",
  "type": "api",
  "config": {
    "method": "POST",
    "url": "{{calendar_api_url}}/events",
    "headers": {
      "Authorization": "Bearer {{calendar_token}}"
    },
    "payload": {
      "summary": "{{event_title}}",
      "start": "{{optimal_start_time}}",
      "end": "{{optimal_end_time}}",
      "attendees": "{{participants}}"
    }
  },
  "dependencies": ["find_optimal_time"]
}

Usage Example‚Äã

curl -X POST http://localhost:8080/api/v1/workflows/calendar_manager/execute \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "context": {
      "schedule_request": "Schedule a 1-hour meeting with John and Sarah next week to discuss the Q4 budget",
      "organizer": "alice@company.com",
      "participants": ["john@company.com", "sarah@company.com"],
      "preferences": {
        "preferred_time": "afternoon",
        "duration": 60,
        "timezone": "America/New_York"
      }
    },
    "started_by": "alice@company.com"
  }'

Customizing Built-in Workflows‚Äã

Extending Workflows‚Äã

You can extend built-in workflows by adding custom steps:

{
  "base_workflow": "email_processor",
  "extensions": {
    "additional_steps": [
      {
        "id": "custom_validation",
        "name": "Custom Email Validation",
        "type": "function",
        "config": {
          "function_name": "custom_email_validator"
        },
        "insert_after": "parse_email"
      }
    ]
  }
}

Configuration Override‚Äã

Override default configurations:

{
  "workflow_id": "file_analyzer",
  "config_overrides": {
    "max_file_size": 10485760,
    "allowed_file_types": ["pdf", "docx", "txt"],
    "security_scanning": true
  }
}

Agent Substitution‚Äã

Replace default agents with custom ones:

{
  "workflow_id": "chat_responder", 
  "agent_mappings": {
    "chat_processor": "custom_chat_agent",
    "intent_analyzer": "advanced_intent_agent"
  }
}

Best Practices‚Äã

Performance Optimization‚Äã

  1. Use Parallel Steps: Leverage parallel execution where possible
  2. Set Appropriate Timeouts: Configure realistic timeouts for each step
  3. Cache Results: Cache expensive operations like file analysis
  4. Resource Monitoring: Monitor agent and system resource usage

Error Handling‚Äã

  1. Graceful Degradation: Provide fallback responses for failures
  2. Retry Logic: Implement retry mechanisms for transient failures
  3. Error Logging: Log detailed error information for debugging
  4. User Feedback: Provide meaningful error messages to users

Security Considerations‚Äã

  1. Input Validation: Validate all inputs before processing
  2. Authentication: Ensure proper authentication for all operations
  3. Data Protection: Protect sensitive data in workflow context
  4. Access Control: Implement proper access controls for workflows

Monitoring and Metrics‚Äã

Built-in Metrics‚Äã

All built-in workflows automatically collect:

  • Execution time per step
  • Success/failure rates
  • Resource utilization
  • Error frequencies
  • User satisfaction scores

Custom Metrics‚Äã

Add custom metrics to workflows:

{
  "metrics": {
    "custom_counters": ["emails_processed", "files_analyzed"],
    "custom_timers": ["analysis_duration", "response_generation_time"],
    "custom_gauges": ["queue_size", "active_conversations"]
  }
}
