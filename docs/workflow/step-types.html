Workflow Step Types

This guide provides comprehensive documentation for all available workflow step types in the Aziron workflow engine.

Basic Communication Steps​

Message Step (message)​

Send a message to a specific agent and wait for response.

{
  "id": "ask_assistant",
  "type": "message",
  "config": {
    "from_agent_id": "user",
    "to_agent_id": "assistant",
    "content": "What is the weather like today?",
    "priority": "normal",
    "metadata": {
      "conversation_id": "{{conversation_id}}"
    }
  },
  "timeout": 30000
}

Configuration Options:

  • from_agent_id: ID of the sending agent
  • to_agent_id: ID of the receiving agent
  • content: Message content (supports variables)
  • priority: Message priority (low, normal, high)
  • metadata: Additional message metadata

Broadcast Step (broadcast)​

Send a message to multiple agents simultaneously.

{
  "id": "notify_all",
  "type": "broadcast",
  "config": {
    "from_agent_id": "system",
    "to_agent_ids": ["agent1", "agent2", "agent3"],
    "content": "System maintenance starting in 10 minutes",
    "priority": "high"
  }
}

Configuration Options:

  • from_agent_id: ID of the sending agent
  • to_agent_ids: Array of receiving agent IDs
  • content: Broadcast message content
  • priority: Message priority
  • timeout: Maximum wait time for all responses

Control Flow Steps​

Conditional Step (conditional)​

Execute different paths based on conditions.

{
  "id": "check_user_role",
  "type": "conditional",
  "config": {
    "condition": "{{user.role}} == 'admin'",
    "true_steps": ["admin_workflow"],
    "false_steps": ["user_workflow"]
  }
}

Configuration Options:

  • condition: Boolean expression to evaluate
  • true_steps: Steps to execute if condition is true
  • false_steps: Steps to execute if condition is false

Supported Operators:

  • Comparison: ==, !=, >, <, >=, <=
  • Logical: &&, ||, !
  • String: contains, startsWith, endsWith
  • Array: in, length

Parallel Step (parallel)​

Execute multiple steps concurrently.

{
  "id": "parallel_analysis",
  "type": "parallel",
  "config": {
    "parallel_steps": [
      "sentiment_analysis",
      "entity_extraction",
      "keyword_analysis"
    ],
    "wait_for_all": true,
    "max_concurrency": 5
  }
}

Configuration Options:

  • parallel_steps: Array of step IDs to execute in parallel
  • wait_for_all: Whether to wait for all steps to complete
  • max_concurrency: Maximum number of concurrent executions
  • timeout: Overall timeout for parallel execution

Sequential Step (sequential)​

Execute steps in a specific order.

{
  "id": "data_pipeline",
  "type": "sequential",
  "config": {
    "sequential_steps": [
      "fetch_data",
      "validate_data",
      "process_data",
      "store_results"
    ],
    "stop_on_error": true
  }
}

Loop Step (loop)​

Repeat steps for a specified number of iterations.

{
  "id": "process_batch",
  "type": "loop",
  "config": {
    "iterations": 10,
    "steps": ["process_item"],
    "break_condition": "{{current_item.status}} == 'complete'",
    "iteration_variable": "batch_index"
  }
}

Delay Step (delay)​

Wait for a specified duration before continuing.

{
  "id": "wait_processing",
  "type": "delay",
  "config": {
    "duration": 5000,
    "unit": "milliseconds"
  }
}

Advanced Control Flow​

Router Step (router)​

Advanced conditional routing with multiple conditions.

{
  "id": "route_request",
  "type": "router",
  "config": {
    "routes": [
      {
        "condition": "{{request.type}} == 'urgent'",
        "steps": ["urgent_handler"]
      },
      {
        "condition": "{{request.category}} == 'support'",
        "steps": ["support_workflow"]
      },
      {
        "condition": "{{request.priority}} > 5",
        "steps": ["high_priority_handler"]
      }
    ],
    "default_steps": ["standard_handler"]
  }
}

Switch Step (switch)​

Multi-way branching based on a single value.

{
  "id": "handle_by_type",
  "type": "switch",
  "config": {
    "switch_on": "{{document.type}}",
    "cases": {
      "pdf": ["pdf_processor"],
      "image": ["image_processor"],
      "text": ["text_processor"],
      "csv": ["csv_processor"]
    },
    "default": ["unknown_type_handler"]
  }
}

Fork Step (fork)​

Split execution into multiple independent branches.

{
  "id": "analysis_fork",
  "type": "fork",
  "config": {
    "branches": [
      {
        "name": "content_analysis",
        "steps": ["extract_content", "analyze_sentiment"]
      },
      {
        "name": "metadata_analysis", 
        "steps": ["extract_metadata", "validate_structure"]
      },
      {
        "name": "security_scan",
        "steps": ["virus_scan", "content_filter"]
      }
    ]
  }
}

Join Step (join)​

Merge results from multiple execution branches.

{
  "id": "merge_results",
  "type": "join",
  "config": {
    "wait_for_branches": ["content_analysis", "metadata_analysis"],
    "merge_strategy": "combine",
    "timeout": 60000
  }
}

While Step (while)​

Execute steps repeatedly while a condition is true.

{
  "id": "process_queue",
  "type": "while",
  "config": {
    "condition": "{{queue.length}} > 0",
    "steps": ["process_next_item"],
    "max_iterations": 100
  }
}

ForEach Step (foreach)​

Iterate over a collection, executing steps for each item.

{
  "id": "process_files",
  "type": "foreach",
  "config": {
    "collection": "{{uploaded_files}}",
    "item_variable": "current_file",
    "index_variable": "file_index",
    "steps": ["validate_file", "process_file"],
    "max_concurrency": 3
  }
}

Data Operation Steps​

Transform Step (transform)​

Transform data using built-in functions or custom code.

{
  "id": "format_response",
  "type": "transform",
  "config": {
    "input": "{{agent_response}}",
    "transformations": [
      {
        "type": "json_parse",
        "field": "content"
      },
      {
        "type": "template",
        "template": "Response: {{content.message}}"
      }
    ]
  }
}

Filter Step (filter)​

Filter data based on specified criteria.

{
  "id": "filter_active_users",
  "type": "filter",
  "config": {
    "input": "{{users}}",
    "condition": "item.status == 'active' && item.last_login > '2024-01-01'"
  }
}

Map Step (map)​

Apply transformations to each item in a collection.

{
  "id": "extract_emails",
  "type": "map",
  "config": {
    "input": "{{contacts}}",
    "transformation": "item.email"
  }
}

Reduce Step (reduce)​

Aggregate data from a collection into a single value.

{
  "id": "calculate_total",
  "type": "reduce",
  "config": {
    "input": "{{orders}}",
    "operation": "sum",
    "field": "amount",
    "initial_value": 0
  }
}

Sort Step (sort)​

Sort a collection based on specified criteria.

{
  "id": "sort_by_priority",
  "type": "sort",
  "config": {
    "input": "{{tasks}}",
    "sort_by": "priority",
    "order": "desc"
  }
}

Group Step (group)​

Group collection items by specified criteria.

{
  "id": "group_by_category",
  "type": "group",
  "config": {
    "input": "{{products}}",
    "group_by": "category"
  }
}

Merge Step (merge)​

Combine data from multiple sources.

{
  "id": "merge_user_data",
  "type": "merge",
  "config": {
    "inputs": [
      "{{user_profile}}",
      "{{user_preferences}}",
      "{{user_history}}"
    ],
    "strategy": "deep_merge"
  }
}

Split Step (split)​

Split data into multiple parts.

{
  "id": "split_text",
  "type": "split",
  "config": {
    "input": "{{large_text}}",
    "method": "chunk",
    "chunk_size": 1000,
    "overlap": 100
  }
}

External Integration Steps​

API Step (api)​

Make REST API calls to external services.

{
  "id": "fetch_weather",
  "type": "api",
  "config": {
    "method": "GET",
    "url": "https://api.weather.com/v1/current",
    "headers": {
      "Authorization": "Bearer {{api_token}}",
      "Content-Type": "application/json"
    },
    "query_params": {
      "location": "{{user_location}}"
    },
    "timeout": 10000
  }
}

Database Step (database)​

Execute database operations.

{
  "id": "save_user",
  "type": "database",
  "config": {
    "operation": "insert",
    "table": "users",
    "data": {
      "name": "{{user.name}}",
      "email": "{{user.email}}",
      "created_at": "{{now}}"
    }
  }
}

File Step (file)​

Perform file system operations.

{
  "id": "read_config",
  "type": "file",
  "config": {
    "operation": "read",
    "path": "/etc/app/config.json",
    "encoding": "utf8"
  }
}

Email Step (email)​

Send email notifications.

{
  "id": "send_notification",
  "type": "email",
  "config": {
    "to": "{{user.email}}",
    "subject": "Workflow Complete",
    "template": "workflow_completion",
    "variables": {
      "workflow_name": "{{workflow.name}}",
      "completion_time": "{{now}}"
    }
  }
}

Webhook Step (webhook)​

Send webhook notifications.

{
  "id": "notify_external",
  "type": "webhook",
  "config": {
    "url": "{{webhook_url}}",
    "method": "POST",
    "payload": {
      "event": "workflow_complete",
      "workflow_id": "{{workflow.id}}",
      "execution_id": "{{execution.id}}"
    }
  }
}

AI & ML Steps​

Embedding Step (embedding)​

Generate text embeddings for semantic search.

{
  "id": "create_embeddings",
  "type": "embedding",
  "config": {
    "text": "{{document.content}}",
    "model": "text-embedding-ada-002",
    "dimensions": 1536
  }
}

Classify Step (classify)​

Classify text into categories.

{
  "id": "classify_content",
  "type": "classify",
  "config": {
    "text": "{{message.content}}",
    "categories": ["spam", "important", "question", "complaint"],
    "model": "classification-model"
  }
}

Summarize Step (summarize)​

Generate text summaries.

{
  "id": "summarize_document",
  "type": "summarize",
  "config": {
    "text": "{{document.content}}",
    "max_length": 200,
    "style": "concise"
  }
}

Translate Step (translate)​

Translate text between languages.

{
  "id": "translate_message",
  "type": "translate",
  "config": {
    "text": "{{message.content}}",
    "from_language": "auto",
    "to_language": "{{user.preferred_language}}"
  }
}

Code Execution Steps​

Code Step (code)​

Execute code blocks in various languages.

{
  "id": "calculate_metrics",
  "type": "code",
  "config": {
    "language": "python",
    "code": "import json\nresult = sum([item['value'] for item in {{data}}])\nprint(json.dumps({'total': result}))",
    "timeout": 30000
  }
}

Script Step (script)​

Execute external scripts.

{
  "id": "run_analysis",
  "type": "script",
  "config": {
    "script_path": "/scripts/analyze_data.py",
    "arguments": ["--input", "{{input_file}}", "--output", "{{output_file}}"],
    "environment": {
      "PYTHONPATH": "/opt/scripts"
    }
  }
}

Function Step (function)​

Execute custom workflow functions.

{
  "id": "custom_processor",
  "type": "function",
  "config": {
    "function_name": "process_data",
    "parameters": {
      "data": "{{input_data}}",
      "options": "{{processing_options}}"
    }
  }
}

Step Configuration Best Practices​

Variable Substitution​

Use double curly braces for variable substitution:

  • {{variable_name}} - Simple variable
  • {{object.property}} - Object property access
  • {{array[0]}} - Array element access
  • {{step_name_response}} - Previous step response

Error Handling​

Configure error handling for robust workflows:

{
  "id": "robust_step",
  "type": "message",
  "config": {
    "retry_count": 3,
    "retry_delay": 1000,
    "fallback_steps": ["error_handler"]
  },
  "on_failure": ["handle_error", "notify_admin"]
}

Timeouts​

Set appropriate timeouts for different step types:

  • Quick operations: 5-10 seconds
  • Agent responses: 30-60 seconds
  • External APIs: 10-30 seconds
  • File operations: 60-300 seconds

Dependencies​

Use dependencies to control execution order:

{
  "dependencies": ["fetch_data", "validate_input"],
  "on_success": ["process_data", "send_notification"],
  "on_failure": ["log_error", "cleanup"]
}
