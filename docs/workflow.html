Workflow System

Aziron's workflow system provides powerful orchestration capabilities for multi-agent AI applications. The workflow engine enables you to create complex, multi-step processes that coordinate between multiple agents, execute conditional logic, and integrate with external systems.

Overview‚Äã

The Aziron workflow system consists of several key components:

  • Workflow Engine: Core execution environment for workflows
  • Workflow Definitions: JSON-based workflow specifications
  • Step Types: Various execution primitives (message, parallel, conditional, etc.)
  • Orchestration Modes: Different coordination strategies
  • Built-in Workflows: Pre-configured workflows for common tasks
  • State Management: Persistent execution state and recovery

Quick Start‚Äã

Creating a Simple Workflow‚Äã

{
  "id": "simple_chat",
  "name": "Simple Chat Workflow",
  "description": "A basic agent conversation workflow",
  "version": "1.0.0",
  "start_step": "send_message",
  "steps": [
    {
      "id": "send_message",
      "name": "Send Message to Agent",
      "type": "message",
      "config": {
        "from_agent_id": "user",
        "to_agent_id": "assistant",
        "content": "{{user_input}}"
      },
      "on_success": ["process_response"],
      "timeout": 30000
    },
    {
      "id": "process_response",
      "name": "Process Agent Response",
      "type": "function",
      "config": {
        "function_name": "format_response"
      }
    }
  ]
}

Executing a Workflow‚Äã

curl -X POST http://localhost:8080/api/v1/workflows/simple_chat/execute \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "context": {
      "user_input": "Hello, how can you help me today?"
    },
    "started_by": "user123"
  }'

Core Concepts‚Äã

Workflow Definitions‚Äã

A workflow definition describes the structure and behavior of a workflow:

  • ID: Unique identifier for the workflow
  • Name: Human-readable name
  • Description: Purpose and functionality description
  • Version: Semantic version for workflow evolution
  • Start Step: Entry point for execution
  • Steps: Array of execution steps
  • Metadata: Additional configuration and tags

Workflow Steps‚Äã

Steps are the building blocks of workflows. Each step has:

  • ID: Unique identifier within the workflow
  • Type: Execution behavior (message, parallel, conditional, etc.)
  • Config: Step-specific configuration parameters
  • Dependencies: Steps that must complete before this step
  • OnSuccess/OnFailure: Next steps based on execution result
  • Timeout: Maximum execution time

Context and Variables‚Äã

Workflows support dynamic context and variable substitution:

{
  "content": "Process this data: {{input_data}}",
  "agent_id": "{{assigned_agent}}",
  "metadata": {
    "correlation_id": "{{correlation_id}}",
    "step_result": "{{step_previous_analysis_response}}"
  }
}

Step Types‚Äã

The workflow engine supports various step types for different execution patterns:

Basic Communication‚Äã

  • message: Send message to a specific agent
  • broadcast: Send message to multiple agents
  • stream: Enable streaming response from agent

Control Flow‚Äã

  • conditional: Execute based on conditions
  • parallel: Execute multiple steps concurrently
  • sequential: Execute steps in sequence
  • loop: Repeat steps with iteration
  • delay: Wait for specified duration

Advanced Control‚Äã

  • router: Advanced conditional routing
  • switch: Multi-way branching
  • fork: Split execution into branches
  • join: Merge execution branches
  • while: Conditional loops
  • foreach: Iterate over collections

Data Operations‚Äã

  • transform: Transform data between steps
  • filter: Filter data based on criteria
  • map: Apply transformations to collections
  • reduce: Aggregate data from multiple sources
  • merge: Combine data from different steps

External Integrations‚Äã

  • api: REST API calls
  • database: Database operations
  • file: File system operations
  • email: Email notifications
  • webhook: Webhook triggers

AI & ML Operations‚Äã

  • embedding: Generate text embeddings
  • classify: Text classification
  • summarize: Text summarization
  • translate: Language translation

Orchestration Modes‚Äã

Aziron supports multiple orchestration strategies:

None (none)‚Äã

  • No response orchestration
  • Raw agent responses returned directly
  • Minimal overhead, maximum performance

Buffer (buffer)‚Äã

  • Collect and buffer streaming responses
  • Return complete responses when finished
  • Suitable for simple aggregation needs

AI Agent (agent)‚Äã

  • Use dedicated AI agent for response orchestration
  • Intelligent analysis and synthesis
  • Advanced reasoning and planning capabilities

Hybrid (hybrid)‚Äã

  • Combine buffering with AI validation
  • Balance performance with intelligence
  • Fallback mechanisms for robustness

Graph (graph)‚Äã

  • Advanced graph-based orchestration
  • Topology analysis and optimization
  • Parallel execution planning
  • Critical path analysis
